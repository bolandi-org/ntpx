# راهنمای کامل راه‌اندازی هسته nptx (مخصوص شبکه‌های دارای محدودیت، CGNAT و Starlink)

این ابزار با تکنولوژی‌های فوق‌پیشرفته شامل NTPv4 Header Mimicry، UDP Spraying (Rain Technique)، و App-Layer Fragmentation ساخته شده تا بر روی مخابراتی‌ترین و بی‌ثبات‌ترین اینترنت‌های جهان (مثل استارلینک یا CGNATهای پشت‌سرهم) با پایداری کامل و بدون افت سرعت اجرا شود.

## ۱. نصب روی سرور خارج (Server)

ابتدا با کاربر روت متصل شوید و اسکریپت نصب را اجرا کنید:

```bash
chmod +x install.sh
sudo ./install.sh
```

اسکریپت در صورت نیاز زبان گولنگ را نصب کرده و باینری را بیلد می‌کند.
سپس فایل کانفیگ `nano /etc/nptx/config.json` را باز کنید:

```json
{
  "mode": "server",
  "local": "0.0.0.0:123",
  "remote": "",
  "streams": 16,
  "password": "یک_پسورد_بسیار_سخت",
  "routes": ""
}
```

**نکته مهم:** مقدار `local` را روی پورت `123` بگذارید، زیرا ترافیک NTP منحصراً مربوط به پورت ۱۲۳ است تا از مسدودسازهای DPI به راحتی عبور کند.
سپس: `systemctl restart nptx`

---

## ۲. نصب روی سرور ایران / کلاینت (Client)

مراحل نصب مشابه سرور خارج است. اسکریپت را اجرا کرده و فایل `nano /etc/nptx/config.json` را برای کلاینت تنظیم کنید:

```json
{
  "mode": "client",
  "local": "",
  "remote": "IP_SERVER_KHAREJ:123",
  "streams": 16,
  "password": "یک_پسورد_بسیار_سخت",
  "routes": "7305:25566,7306:51820"
}
```

### توضیح بخش Routes (اتصال‌ها)

در بخش `routes`، شما مشخص می‌کنید که کدام پورتِ داخلی در سرور ایران (کلاینت)، باید به چه پورتی در سرور خارج وصل شود!
به عنوان مثال `7305:25566` یعنی:
«کلاینت nptx روی پورت لوکال (ایران) `7305` گوش بدهد، و هر دیتایی دریافت کرد را با حداکثر سرعت و تکه-تکه‌شده (Fragmentation) زیر پوشش پکت‌های NTP برای سرور خارج بفرستد تا سرور خارج پس از دریافت، آن‌ها را رمزگشایی و دقیقاً بازسازی کرده و به پورت `25566` در داخل سرور خارج تحویل دهد.»

سپس: `systemctl restart nptx`

## ۳. تست یک سناریوی کامل با WireGuard

فرض کنید قصد دارید ترافیک وایرگارد را از ایران به خارج تونل بزنید.
۱- وایرگاردِ سرور خارج را تنظیم کنید تا روی پورت `51820` فعال شود.
۲- در سرور ایران، کانفیگ nptx کلاینت را باز کرده و `routes` را اینطور وارد کنید: `"51820:51820"`. (یعنی پورت 51820 ایران وصل بشه به 51820 خارج).
۳- سیستم‌دلِی (وایرگاردِ روی کلاینت‌ها/موبایل دوستانتان) را تنظیم کنید تا به Endpoint: `IP_IRAN:51820` متصل شوند.

**فرآیند جریان ترافیک:**
موبایل → (Wireguard 1420 MTU) → سرور ایران ۱ (پورت ۵۱۸۲۰) → (هسته nptx ایران) → تکه‌شدن پکت 1420 به سایز امن 1124 → دریافت هدر NTPv4 دروغین → اسپری شدن روی ۱۶ سوکت موازی → استارلینک / DPI → سرور خارج (پورت ۱۲۳) → جمع‌آوری سوکت‌ها → تایید هویت ChaCha20 → چیدن دوباره قطعات از هم‌گسسته با تحمل خطای ۲۰ ثانیه‌ای → تحویل پکت کامل ۱۴۲۰ بایتی به وایرگارد روی پورت ۵۱۸۲۰!

همه چیز به صورت جادویی و کاملاً اتوماتیک کار خواهد کرد. لذت ببرید!
